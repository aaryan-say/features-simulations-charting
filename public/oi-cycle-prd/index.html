OI Positioning Cycle

1Summary

A rotation graph for F&O traders. It uses the same visual as a Relative Rotation Graph (a scatter with four quadrants and trailing tails), but the axes are the two things F&O positioning turns on: price change (X) and open-interest change (Y). Each instrument is plotted by how its price and OI moved over the period, and its tail shows how it has rotated through the positioning regimes over the last few periods.

The four quadrants are the standard F&O read: Long Buildup (price up, OI up), Short Buildup (price down, OI up), Long Unwinding (price down, OI down), Short Covering (price up, OI down). Scoped to options / F&O only (OI is a derivatives concept; cash equities have none).

What it tells a trader

  • Long Buildup: fresh longs, bullish conviction building.
  • Short Buildup: fresh shorts, bearish conviction building.
  • Long Unwinding: longs exiting, bullish conviction fading.
  • Short Covering: shorts exiting, often a squeeze / bounce.
  • The tail is the edge: watching an instrument rotate (e.g. Short Buildup then into Short Covering) flags a shift before the label does.

3What the simulation demonstrates

The Live Cycle tab is a working proof-of-concept: several F&O names plotted by (price change, OI change), each with a rolling tail, classified into the four quadrants live, with the side table naming each instrument's current state. It runs the real classification and tail logic; only the raw price/OI moves are synthetic.

7The ask to backend

Request (small)

This feature is almost entirely client-side on data that exists. One verification:

  • OI history series. Confirm cumulative_oi / cumulative_fut_oi return a daily/weekly series per F&O instrument (needed for the tails). Live OI change is already available; if the history returns, there is no backend build for this feature.
  • Confirm the OI level used. Which OI to represent an underlying: aggregate F&O OI, futures-only OI, or ATM-band option OI. Affects how a name is classified.

8Open questions

  • Does OI history come back at daily and weekly for the tails, and how far back?
  • Per-underlying OI definition (futures vs aggregate vs option-band)?
  • Expiry handling: OI resets/rolls at expiry, which can distort period-over-period OI change. Roll-adjust, or note it?
  • Universe: fixed F&O list, index constituents, or the user's watchlist?

9Risks & limitations

  • OI is F&O positioning intent, not a price prediction; the quadrants describe what positioning is doing, not a signal to act on blindly.
  • Expiry rollover distorts OI change (OI migrates to the next series); needs roll-adjustment or a clear caveat.
  • Thin F&O names have noisy OI; classification can flip on small moves.
  • Dense graph is desktop-first; mobile needs a rethink.
