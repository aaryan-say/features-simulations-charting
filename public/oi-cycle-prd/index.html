Options Positioning Cycle

1Feature description

A live rotation graph that plots a trader's chosen option strikes by their price change (X) and open-interest change (Y) into four positioning quadrants, with a trailing tail per strike showing how it moves between states over time. It answers what an option-chain snapshot cannot: not just "where is positioning" but "which way is it rotating."

The four quadrants (neutral backgrounds, corner labels):

  • Long Buildup (price up, OI up) · top-right
  • Writing (price down, OI up) · top-left
  • Long Covering (price down, OI down) · bottom-left
  • Short Covering (price up, OI down) · bottom-right

Each strike is a marker: a dot coloured by type (green = call, red = put), with a small badge showing A / O / I (At / Out / In-the-money, relative to live spot, so it updates), the strike label, and a trailing tail. One underlying + expiry per cycle. It opens with a default of 4 strikes (ATM call, ATM put, the highest-OI OTM call = resistance, the highest-OI OTM put = support); the user adds or removes strikes up to ~8-10.

2Use case

An options trader wants a fast read of where positioning is forming and, more importantly, where it is shifting across the specific strikes they trade, without scanning OI numbers row by row. Examples:

  • See a call strike sitting in Writing = resistance being defended; watch it rotate into Short Covering = writers capitulating, often ahead of an up-move.
  • See puts rotating into Writing = support forming under the market.
  • Track their own strategy legs (a specific call + put) rotating together, to manage or exit.

Opened intraday; the value is the movement, not a static tag.

4Frontend instructions

  • Layout: filter bar (underlying, expiry, interval, tail, add-strike) + rotation graph + side table.
  • Classification: quadrant = signs of (price change, OI change): (+,+) Long Buildup, (−,+) Writing, (−,−) Long Covering, (+,−) Short Covering.
  • Moneyness A/O/I: compute vs live spot. Call: strike > spot = OTM, < spot = ITM; Put: strike < spot = OTM, > spot = ITM; near spot = ATM. Recompute each tick so it updates as price moves.
  • Markers: dot colour by type (green call / red put); the moneyness letter sits inside the marker; strike label beside; tail = polyline of the last N points in the same colour.
  • Quadrants: neutral grey backgrounds; identity comes from position + corner labels, not colour, so the green/red dots stay legible.
  • Directional read: map (type, quadrant) to plain text in the tooltip / side table, since colour encodes type not bias (a green call in Writing is bearish/resistance).
  • Cap & defaults: enforce the strike cap; favour liquid near-ATM strikes in defaults (thin strikes give short/noisy tails).

5Backend instructions

  • Per strike, per period: the option's price and OI. Available from the option chain (/optionchains/:asset?exchange=NSE&expiry=), which returns oi, prev_oi, ltp, ltpchg per strike. OI change = oi − prev_oi; price change = ltpchg.
  • Spot (for moneyness) from the underlying LTP / current price.
  • Tails: per-option OI + price history at the interval (per-option timeseries). Options are short-lived, so the tail is bounded by the strike's age; favour near-expiry / active strikes.
  • Reuse if available: the backend already classifies per-strike positioning (the OI Trader "opportunities" surface tags Long Buildup / Writing / Covering). If that classification (and its history) can be exposed, the frontend can consume it directly instead of recomputing.
  • Coverage: options only. Delivery via the market feed for live updates, or the option-chain endpoint polled at the interval.
